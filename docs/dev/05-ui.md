# 05 - UI (Web App)

## Overview

React SPA for managing KeepAI: connecting services, managing agents, configuring
policies, approving requests, and viewing logs.

Served by keepd on localhost. Embedded in electron. Tech stack same as ../keep.ai
web app: React + Vite + TailwindCSS + Radix UI + TanStack Query.

## Pages

### Dashboard (`/`)
- Pending approvals at the top (if any) — same approve/deny cards as `/approvals`
- Full list of connected services with icons, account names, status
- Full list of paired agents with status (online/offline), last seen
- Quick actions: "Connect service", "Add agent"

### Connections (`/connections`)
- List of connected service accounts
- Each shows: service icon, account name/email, status (connected/error), last used
- Single "Connect service" button → opens **service picker dialog** with list of
  supported services (Gmail, Notion for now; designed to scale to hundreds later)
- Disconnect button per account
- "Test" button to verify connection health

### Connection Detail (`/connections/:service/:accountId`)
- Account details
- Connection status and last refresh time
- Usage stats (requests via this account)
- Disconnect action

### Agents (`/agents`)
- List of paired agents
- Each shows: name, status (online/offline), last seen, request count
- "Add Agent" button → opens pairing dialog
- Revoke button per agent

### Agent Detail (`/agents/:agentId`)
- Agent info (name, pubkey, paired date, last seen)
- Per-service policy summary
- Recent requests by this agent
- Revoke action

### Pairing Dialog (modal)
1. Enter agent name (e.g., "openclaw")
2. Click "Generate Code"
3. Shows pairing code + instructions:
   ```
   Tell your agent to run:
   npx keepai init <code>
   ```
4. Waits for agent to complete pairing (real-time via SSE)
5. Shows success when agent pairs

### Policies (`/agents/:agentId/policies`)
- Per-service policy cards
- For each service:
  - Default action: allow / deny / ask (dropdown)
  - Operation rules:
    - Read operations: allow / deny / ask
    - Write operations: allow / deny / ask
    - Delete operations: allow / deny / ask
  - Raw JSON editor toggle (for power users)
- Save button
- "Reset to defaults" button

### Approval Queue (`/approvals`)
- List of pending approval requests
- Each shows:
  - Agent name + icon
  - Service + method
  - Account
  - Human-readable description (e.g., "Send email to bob@example.com")
  - Time waiting
  - [Approve] [Deny] buttons
- Real-time updates via SSE (new items appear automatically)
- Expandable request details (full params, preview)
- Badge count in sidebar navigation

### Logs (`/logs`)
- Searchable/filterable table of all past requests
- Columns: timestamp, agent, service, method, account, status, policy action, duration
- Filters: by agent, by service, by date range, by status
- Click row to expand and see full request/response details

### Settings (`/settings`)
- Daemon configuration:
  - Nostr relay URLs
  - Approval timeout duration
  - OAuth callback port (for Docker scenarios)
- About section (version, links)

## State Management

### Approach: TanStack Query + HTTP API

No crsqlite. All state lives in keepd's SQLite database, accessed via HTTP API.

```typescript
// hooks/useConnections.ts
export function useConnections() {
  return useQuery({
    queryKey: ['connections'],
    queryFn: () => fetch('/api/connections').then(r => r.json()),
  });
}

// hooks/useAgents.ts
export function useAgents() {
  return useQuery({
    queryKey: ['agents'],
    queryFn: () => fetch('/api/agents').then(r => r.json()),
  });
}

// hooks/useApprovalQueue.ts
export function useApprovalQueue() {
  return useQuery({
    queryKey: ['queue'],
    queryFn: () => fetch('/api/queue').then(r => r.json()),
  });
}

// Mutations
export function useApproveRequest() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => fetch(`/api/queue/${id}/approve`, { method: 'POST' }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['queue'] });
    },
  });
}
```

### Real-time Updates via SSE

```typescript
// hooks/useSSE.ts
export function useSSE() {
  const queryClient = useQueryClient();

  useEffect(() => {
    const source = new EventSource('/api/events');

    source.addEventListener('approval_request', () => {
      queryClient.invalidateQueries({ queryKey: ['queue'] });
    });

    source.addEventListener('pairing_completed', () => {
      queryClient.invalidateQueries({ queryKey: ['agents'] });
    });

    source.addEventListener('agent_connected', () => {
      queryClient.invalidateQueries({ queryKey: ['agents'] });
    });

    source.addEventListener('request_completed', () => {
      queryClient.invalidateQueries({ queryKey: ['logs'] });
    });

    return () => source.close();
  }, [queryClient]);
}
```

SSE events trigger TanStack Query invalidation → automatic refetch → UI updates.

## Vite Build Modes

### `build:frontend` (default)
- Output: `dist/frontend/`
- API base: `/api` (relative, served by keepd)
- Used when keepd serves ui directly

### `build:electron`
- Output: `dist/electron/`
- API base: read from `window.env.API_ENDPOINT` (set by electron preload)
- Relative asset paths for electron

No serverless mode (unlike keep.ai — we always need keepd).

## Electron-specific Features

When running in electron:
- Approval requests also trigger OS-level desktop notifications
- Clicking notification opens ui to `/approvals`
- IPC via preload: `window.electronAPI.showNotification()`
- Tray icon badge shows pending approval count

```typescript
// In App.tsx or layout component
useEffect(() => {
  if (window.electronAPI) {
    // Update tray badge when approval count changes
    window.electronAPI.updateTrayBadge(pendingCount);
  }
}, [pendingCount]);
```
